Authentication Contract

Studio UI authenticates against `rulex-identity-api`, then calls `rulex-studio-api` with bearer tokens and correlation IDs. Both services validate JWT with shared middleware from `rulex-common-lib`.

Authorization: Bearer <access_token> x-correlation-id: <session-correlation-id>
OpenAPI: `rulex-studio-api` Swagger v1 (OAS 3.0) at
https://qhh3p934pv.us-west-2.awsapprunner.com/swagger/v1/swagger.json

Response & Error Model

APIs use middleware-driven error handling, and domain services throw rule-specific exceptions. UI callers should parse non-OK payloads (text/JSON) and surface actionable messages.

{ "message": "Descriptive error message", "statusCode": 400 }

Operational Notes

  • Always send bearer token for authenticated routes, including logout.
  • Use ruleversion endpoints for version downloads and comparisons.
  • Keep route usage case-sensitive and aligned with server paths.
  • Treat Test endpoints as diagnostic/debug routes, not primary business APIs.
