These repositories form the foundation of the Rule-X platform and are free for anyone to use, modify, and contribute to.

rulex-engine
MIT
The core rule evaluation engine. Executes Rule-X rule graphs against input payloads. Supports all 13 block types including CEP, Temporal, and Inference Engine. Zero external dependencies.
npm install @rulex/engine

rulex-sdk-js
MIT
Official JavaScript/TypeScript SDK for the Rule-X REST API. Full TypeScript types, retry logic, pagination helpers, and real-time evaluation WebSocket support.

rulex-temporal
Apache 2.0
Standalone TypeScript library implementing all 13 of Allen's interval relations for temporal logic. Can be used independently of Rule-X for any project that needs interval-based time reasoning.

Philosophy

Why we believe in
open source

🔍

Transparency

Business rules should be transparent and auditable. Open sourcing our evaluation engine means you can verify exactly how your rules are executed — no black boxes.

🤝

Community

The best software is built collaboratively. We actively invite contributions, fix bugs reported by the community, and give credit where it's due.

🚫

No Vendor Lock-in

Your rule data is yours. The open source engine means you can always run your rules locally, migrate to another system, or self-host — without asking our permission.

📖

Open Standards

We implement DMN (Decision Model and Notation) as an open standard, and we contribute to the OMG working group for temporal and CEP rule standards.
