Developer Hub Overview

This page now uses a Harness-style documentation layout with content mapped to your real services, repos, and endpoint behavior. It is designed as the central architecture hub for onboarding and technical handoff.

Real repo grounding: `rulex-studio-ui-main`, `rulex-identity-api`, `rulex-studio-api`, and `rulex-common-lib`.

Repo Map

🖥️ rulex-studio-ui-main 🔐 rulex-identity-api ⚙️ rulex-studio-api 📦 rulex-common-lib
Studio UI (React + Vite)
Owns auth state, feature navigation, and API client orchestration for rule authoring, pull requests, and deployments.
Identity API (.NET 8)
Handles login, refresh, logout, user token claims, correlation-id propagation, and refresh-token persistence.
Studio API (.NET 8)
Provides organization, domain, project, rule, ruleversion, execution, deployment, environment, and pull-request APIs.
Common Library Contracts
Centralized JWT middleware, authentication handler, logging middleware, and RuleX exception model.

End-to-End Request Flow

1
Login Request from UI

UI sends POST /authentication/login to identity-api.

↓
2
Token Response

identity-api returns accessToken, refreshToken, and correlationId.

↓
3
Authenticated Studio Requests

UI stores tokens and sends bearer auth + correlation header to studio-api calls.

Authorization: Bearer <access_token> x-correlation-id: <session-correlation-id>
↓
4
JWT Validation in API Layer

studio-api validates JWT using shared middleware from rulex-common-lib.

↓
5
401 Recovery + Retry

On 401, UI calls /authentication/refresh, updates tokens, and retries once.

Primary Platform Capabilities

  • Rule lifecycle: create, version, commit, compare, revert, and archive.
  • Deployment lifecycle: deploy, history, rollback, deactivate, and compare versions.
  • Review lifecycle: pull request approvals, merge, comments, and environment-aware release flow.
  • Security model: JWT-based auth, correlation ID tracing, and shared middleware contracts.

Integration Notes (High Priority)

  • Logout should always send bearer token when calling `POST /authentication/logout`.
  • Rule download should align to ruleversion download endpoint behavior.
  • Keep UI route usage and studio-api ruleversion route semantics strictly aligned.
  • If UI uses both studio-api and Supabase directly, enforce one data source per workflow to avoid drift.
